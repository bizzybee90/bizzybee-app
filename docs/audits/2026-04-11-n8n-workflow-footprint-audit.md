- `GOOD_AGENT_CANDIDATE`: multi-step judgment/synthesis/extraction where agent orchestration is likely to improve quality and adaptability.

## Highest-Priority First Migration Target

`faq_generation` is the first migration target.

Why first:

- It is the strongest reasoning/synthesis fit among live n8n flows.
- It is bounded (known competitor input set, known FAQ output contract).
- It is less safety-critical than email classification and less orchestration-heavy than full competitor discovery.
- Existing guidance already points here first: [`docs/HANDOFF-2026-04-11-CLAUDE-AGENTS.md:357`](../HANDOFF-2026-04-11-CLAUDE-AGENTS.md#L357).

## Key Audit Findings

1. Core n8n orchestration is centralized in one edge function (`trigger-n8n-workflow`) with four supported workflow types (`competitor_discovery`, `email_classification`, `own_website_scrape`, `faq_generation`).
2. `n8n-competitor-callback` currently handles multiple workflow families (competitor discovery, competitor scrape status, and own website FAQ writes), indicating coupling that should be split as migrations proceed.
3. Repo still has stale/misaligned callers against the trigger contract:
   - [`EmailAccountCard.tsx:162`](../../src/components/settings/EmailAccountCard.tsx#L162) sends `workflow_type: 'email_import'` and no `workspace_id`, but trigger requires `workspace_id` and does not implement `'email_import'`.
   - [`TriageQuickActions.tsx:44`](../../src/components/conversations/TriageQuickActions.tsx#L44) and [`LowConfidenceWizard.tsx:122`](../../src/components/settings/LowConfidenceWizard.tsx#L122) call `trigger-n8n-workflow` expecting synchronous re-triage result objects; current trigger path is async dispatch only.

## Residual Risks

1. Environment contract risk: both `trigger-n8n-workflow` and `elevenlabs-webhook` read `N8N_WEBHOOK_URL`, but one expects a base URL while the other appears to expect a full endpoint URL.
2. Hard-coded endpoint risk: FAQ fanout URL is hard-coded in [`n8n-competitor-callback/index.ts:9`](../../supabase/functions/n8n-competitor-callback/index.ts#L9), creating divergence from env-based config.
3. Observability gap: the documented GDPR scheduled workflow is external to app runtime tracking and not represented as app-owned run records.
4. Callback coupling risk: one callback function (`n8n-competitor-callback`) currently multiplexes multiple lifecycle domains, increasing blast radius for changes.
