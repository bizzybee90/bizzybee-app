## 1. COMPLETED: Facebook Messenger Webhook Bug Fixes

**Files changed:**

- `supabase/functions/facebook-messenger-webhook/index.ts`
- `supabase/functions/instagram-webhook/index.ts`

**4 bugs fixed in both webhooks (deployed):**

1. **Duplicate message protection** — Meta retries webhook delivery on timeouts. Added graceful handling of unique constraint violation (error code `23505`) on `messages(conversation_id, external_id)`. Duplicates now skip silently instead of crashing.

2. **Conversation status preservation** — Every inbound message was resetting conversation status to `new`, even if an agent had moved it to `escalated` or `ai_handling`. Fixed: only new conversations get `status: 'new'`; existing conversations just update `requires_reply` and timestamps.

3. **Customer creation race condition** — Added unique index `customers_workspace_phone_uidx` on `(workspace_id, phone) WHERE phone IS NOT NULL`. Code now uses upsert with conflict handling + fallback re-fetch.

4. **Signature verification required** — If `META_APP_SECRET` is not set, webhooks now return 500 instead of silently accepting unsigned payloads.

**Migration applied:** `add_customers_workspace_phone_unique`

**Fix (Meta App Dashboard, not code):**

1. Went to Instagram API → Use Cases → API setup with Instagram login → Added all required messaging permissions
2. Went to API setup with Facebook login → Added permissions from both sections
3. This unlocked `pages_read_engagement` in the FLfB config editor
4. Added `pages_read_engagement`, `instagram_basic`, `instagram_content_publishing`, `instagram_manage_comments`, `instagram_manage_messages` to the original FLfB config (`2081766425888659`)

**Code changes (deployed):**

- `supabase/functions/meta-auth-callback/index.ts` — Added 3-method Instagram discovery fallback:
  - Method A: `/{pageId}?fields=instagram_business_account` (needs `pages_read_engagement`)
  - Method B: `/me/businesses` → `/{biz_id}/instagram_accounts` + `owned_instagram_accounts` (needs `business_management`)
  - Method C: `/{pageId}/page_backed_instagram_accounts` (alternative endpoint)
- Also added system-user token handling (skips long-lived exchange if token doesn't expire)

**Result:** @maccleaningservices (ID: 17841402954772290) auto-discovered and connected via single "Connect with Facebook" click.

**New edge function created:** `supabase/functions/meta-sync-channels/index.ts` — utility to re-check Instagram info using stored Page Access Token (deployed, useful for debugging).
